Visão Geral

A linguagem de programação C foi criada por Dennis Ritchie em 1972. É uma linguagem de programação versátil para sistemas inteligentes. Além de possuir uma tipagem forte e estática, é uma linguagem procedural com nível de abstração intermediário, ou seja, não é orientada a objetos e possui recursos mais próximos ao hardware.

Hello World

Toda linguagem de programação possui funções ou comandos nativos responsáveis por auxiliar o desenvolvedor na criação de suas aplicações.
As primeiras funções, e mais básicas, são aquelas que nos permite mostrar informações. Em C, temos várias, mas as funções principais sãos printf e puts. 

#include <stdio.h>
int main(void)
{
    puts("Hello, World!");
    return 0;
}

O algoritmo acima possui uma função principal chamada main responsável por executar o nosso programa. Ela tem um valor de retorno indicado pelo int em sua assinatura, por convenção, sempre será zero (por isso o comando return 0). Além disso, nossa função não receberá parâmetros devido ao void. 

int main(void)

Parâmetro: É uma ou mais informações passadas para a função. Com elas, a função pode fazer algum tipo de processamento e retornar um resultado.

Retorno: É um resultado gerado através do processamento das informações passadas por parâmetros.

Outro comando que pode ficar na duvida é a diretiva de pré-processamento #include <stdio.h>. Ela é apenas um cabeçalho com um monte de assinaturas de funções internas da linguagem C. É por meio desse comando que "chamamos" as funções input e output, ou seja, funções de entrada e saída de dados.

Abaixo, temos por fim, a instrução responsável por mostrar uma mensagem no terminal: 

puts("Hello, World!")

Outra forma de mostrar mensagens no terminal seria: 

printf("Hello, World!")

A diferença entre puts e printf, é que a primeira só mostra mensagens definidas, enquanto a segunda nos permite formatar os dados antes de serem exibidos. Algumas formatações incluem a interpolação com espaçamentos, precisão de valor, alinhamento de texto, maior controle de escapes e exibições explicitas com flags.

Tipificando as Informações

Toda informação, quando bem tratada, possui uma classificação conforme o conjunto de características ou comportamentos que a ele possui, por exemplo, com números, conseguimos fazer operações aritméticas, mas com letras ou frases, não. Esse é um dos motivos que sustentam o fato de haver tipos primitivos nas linguagens de programação. Em C, há três tipos principais:

  • int: Para armazenar números inteiros.
  • float: Para armazenar números reais.
  • char: Para armazenar caracteres únicos.

Para armazenar e manipular informações devemos criar variáveis precedidas pelos tipos primitivos. Sua sintaxe é relativamente simples, só devemos color o tipo de dado, um nome para a variável e a informação que ela recebe. 

int idade = 20;
float dinheiro = 12.34f;
char sinal = '$';

Para mostras esses dados com as funções de saída, ficará algo como: 

printf("int: %i", idade)
printf("float: %f", dinheiro)
printf("char: %c", sinal)

Repare que o caractere de formatação %... é responsável por captar a informação contida na variável e interpolar na mensagem de exibição. Além disso, note que o caractere corresponde ao tipo primitivo da variável.

Captando Dados

A simples declaração de nossas variáveis e a sua devida inicialização ainda mantêm nosso programa estático, para que os dados mudem a cada execução, devemos usar funções de entrada que irão armazenar dados nas variáveis conforme o que é passado. Na linguagem C, cada tipo primitivo recebe uma função própria, com exceção de int e float que são dados numéricos. 

scanf("%i", &idade)
scanf("%f", &dinheiro)
sinal = getchar()

Vale ressaltar que, antes de usar as funções de entrada, é necessário declarar as variáveis.

A função scanf recebe um dado formatado e coloca ele no endereço de uma variável (para acessar o endereço, devemos colocar um & antes de qualquer identificador). Já a função getchar permite fazer uma atribuição direta para uma variável.

Operações

Talvez essa seja a seção mais relevante, pois aqui será abordado manipulações simples entre dados numéricos, caracteres e, inclusive, lógicos.

Toda informação pode ser tratada e processada. Valores numéricos, por exemplo, podem submetidos à operações aritméticas. Já os caracteres, podemos fazer verificações como dados lógicos.

Tipo Operação Exemplo
Números
#include <math.h> 		abs(int) abs(-5) = 5
fabs(double) fabs(-5.7) = 5.7
ceil(double) ceil(5.5) = 6.0
floor(double) floor(5.5) = 5.0
sqrt(double) sqrt(49.0) = 7.0
pow(double, double) pow(2.0, 5.0) = 32.0
fmod(double, double) fmod(5.0, 2.0) = 1.0
Caracteres
#include <ctype.h> 		isdigit(int) isdigit('A') = 0
isalpha(int) isalpha('A') = 1
isalnum(int) isalnum('8') = 4
isspace(int) isspace(' ') = 8
ispunct(int) ispunct('!') = 16
toupper(int) toupper('x') = 'X'
tolower(int) tolower('X') = 'x'

As funções para o tratamento de caracteres sempre irão retornar um valor igual a 0 se a condição for falsa, caso contrário, o retorno será um valor positivo, geralmente representado por uma potência na base 2.

Em programação, o valor 0 pode ser interpretado como falso e 1 como verdadeiro, ou seja, uma função sempre retornará um valor diferente de 0 para indicar verdadeiro o resultado do processamento.

Pelo exemplo acima, é necessário entender outro tipo primitivo, nesse caso, são as proposições lógicas. Em geral, há apenas dois resultados para uma proposição: verdadeiro (TRUE) ou falso (FALSE), além disso, uma proposição pode ser submetida por um processamento através dos operadores lógicos, eles são úteis para englobar mais de uma condição, deixando nosso algoritmo cada vez mais completo e próximo da realidade.

Em C, não há o tipo lógico, mas algumas API's, como a do Windows, por exemplo, nos permite usar definições para trabalhar com dados lógicos, no entanto, estaremos trabalhando com valores numéricos de qualquer jeito, sem contar que podemos nós mesmos definir diretivas de pré-processamento para "criar" esse tipo de dado. A declaração desses dados lógicos com a API do Windows ficará algo como: 

BOOL F = FALSE;
BOOL T = TRUE;

Como mencionado anteriormente, enquanto ao tratamento desso tipo primitivo, possuímos três operadores principais: 

!TRUE = FALSE
!FALSE = TRUE

TRUE && TRUE = TRUE
TRUE && FALSE = FALSE
FALSE && TRUE = FALSE
FALSE && FALSE = FALSE

TRUE || TRUE = TRUE
TRUE || FALSE = TRUE
FALSE || TRUE = TRUE
FALSE || FALSE = FALSE

Repare que o operador de negação (!) é unário, isto é, que depende apenas de um único operando, já os operadores de conjunção (&&) e disjunção inclusiva (||) dependem de duas proposições, ou seja, são operadores binários. Enquanto aos resultados desses operadores, podemos afirmar que a negação apenas troca o valor da proposição ao seu oposto. Já a conjunção, sempre retornará verdadeiro se ambas as proposições forem verdadeiras, para que isso se repita na disjunção inclusiva, basta uma das proposições ser verdadeira para que o resultado seja verdadeiro.

Tomadas de Decisão

Os comandos até então discutidos, criam, restritamente, uma programação linear, no entanto, haverá casos que nosso algoritmo deve ser capaz de decidir ações conforme uma operação ou entrada de dados. Por exemplo, a criação de uma simples aplicação para calcular a média de um aluno e mostrar se ele é aprovado ou não, dependerá da sua nota, se para passar de ano ele deve ter 7.0 pontos ou mais, o nosso programa mostrará uma mensagem dizendo que ele foi aprovado caso tenha atingido a média, senão, a mensagem informará que ele está em recuperação ou reprovado. Para que esse tipo de comportamento seja realizado pelo programa, devemos usar estruturas de decisão. 

if (nota >= media) {
  puts("Aprovado");
} else {
  puts("Reprovado");
}

Se a condição do comando if for verdadeira, será mostrado "Aprovado", caso contrário, o bloco de instruções do comando else será executado. Agora, imagine se o aluno tirou entre a 6.9 e 5.0, isso permitiria que colocássemos uma segunda condição para indicar que ele pode fazer uma prova substitutiva. Para isso, devemos aninhar uma segunda condição do tipo else if, assim permitindo um terceiro comportamento do nosso programa. 

if (nota >= media) {
  puts("Aprovado");
} else if (nota < media && nota >= 5.0) {
  puts("Fazer Substitutiva!");
} else {
  puts("Reprovado");
}

Apesar de ser dispensável o seu uso em verificações simples como acima, o operador lógico de conjunção foi útil para verificar se a nota esta abaixo da média e é maior ou igual que 5.0 pontos. Caso uma das condições seja falsa, o programa irá executar o bloco do comando else.

Ciclos

Haverá momentos que seremos obrigados a repetir determinados comandos. Imagine que queremos mostrar todos os múltiplos de 7 entre 0 e 100. Para tarefas desse gênero, devemos usar um ciclo irá repetir constantemente instruções pré-definidas, nesse caso, mostrar todos os múltiplos de 7 entra o intervalo de 0 e 100.

Toda estrutura de repetição (ciclo), deve ter um valor de partida (variável de controle), valor de chegada (condição de parada) para interromper o ciclo e um incremento para saltar de um valor até o outro, fazendo com que a condição de parada seja alcançada em algum momento. 

for (int cont = 0; cont <= 100; cont++)

Repare que a variável cont é nosso contador que vai de 0 até 100. Para mostrar apenas os múltiplos de 7, devemos colocar em desvio condicional que mostrará apenas esse valores.

Todo múltiplo de 7 quando divido pelo mesmo valor, terá o resto da divisão igual a 0. Por exemplo, quando dividimos 14 por 7 o resultado será 2 e o módulo 0. Agora se dividimos 15 por 7 o resultado será 2,14 e o módulo 1, ou seja, verdadeiro na programação. Na realidade, qualquer número diferente de zero é verdadeiro.

Concluímos que: todo número quando não for divisível por 7 irá retornar um valor qualquer diferente de 0, esse valor qualquer é interpretado como verdadeiro em uma condição, mas como queremos apenas os valores que são múltiplos, isto é, que possuem o módulo igual a 0 (falso em uma condição), podemos fazer uma inversão desse valor lógico. 

for (int cont = 0; cont <= 100; cont++)
{
    if (!fmod(cont, 7))
    {
        printf("%i\n", cont);
    }
}

Desse modo, concluímos a proposta do exemplo mencionado anteriormente. Vale ressaltar que o uso de ciclos é algo extremamente importante quando queremos automatizar algumas tarefas simples.
